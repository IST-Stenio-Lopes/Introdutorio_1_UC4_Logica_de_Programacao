Utilizando as estruturas de dados, quais estratégias Felipe pode usar para solucionar o problema da loja?

Nesta Unidade de Estudo, você conhecerá os conceitos fundamentais sobre filas, pilhas e listas. Já é importante saber que umas das principais semelhanças entre esses três tipos de estruturas de dados é que cada dado armazenado, em qualquer uma delas, é chamado de Elemento, e assim também tratado nas principais linguagens, como em C# que é a que estamos utilizando como exemplo.

Dependendo da linguagem de programação utilizada, a maneira de criação e manipulação (ou tratamento) de uma fila, pilha ou lista pode mudar, mas a lógica por trás continua a mesma. Então é recomendado conhecer alguns pontos importantes, que são:

  • Ao criar uma estrutura de dados, é preciso definir de que tipo são seus elementos(int, float, string e etc...) e dar um nome para ela, seguindo as mesmas regras de nomeação de uma variável;
  • Instanciar a estrutura (para linguagens de programação orientadas a objetos como Java e C#, por exemplo) ou seja, declarar que ela é um novo objeto sendo criado;
  • Saber acessar elementos da estrutura, para também remover elementos ou adicionar novos.

    • Vamos começar conhecendo o que é uma fila.

    Filas

    Uma fila adota o conceito conhecido como FIFO, do inglês First In, First Out, ou seja, o primeiro dado que entra será o primeiro a sair. Fazendo uma comparação com o nosso dia a dia, imagine que você tenha que entrar numa fila de supermercado. Você deverá obedecer a ordem de chegada das pessoas. Sendo assim, a primeira pessoa que chegou na fila será a primeira a ser atendida. Outro exemplo de fila, é a sequência das impressoras, que imprimem um documento conforme ordem de chegada, definida por um comando emitido pelo computador, notebook, ou algum dispositivo móvel conectado.

    Pilhas

    Quem nunca teve uma “pilha” de livros no canto do quarto, ou aquela “pilha” de roupas para lavar? Ou até mesmo aquela “pilha” de exercícios que o professor passou para fazer? Em programação, uma pilha é uma estrutura de dados que possui uma lógica chamada LIFO, do inglês Last In, First Out. Em português, último a entrar, primeiro a sair. Isso quer dizer que, em termos lógicos, o último dado que for inserido na pilha será o primeiro a ser removido. Sendo assim, os dados de entrada e saída serão disponibilizados na mesma ponta da pilha, diferente das condições de uma lista, como vimos anteriormente. Segue um exemplo para melhor compreensão:

    O conceito de pilha é utilizado em processadores de texto, como o Microsoft Word, por exemplo, quando você quer desfazer ou refazer um texto. Ou ainda, quando está navegando em páginas na internet e deseja retornar à página anterior e, até mesmo, em programas como calculadoras.     Listas

    Imagine que você precisa ir ao supermercado fazer as compras, e quer saber quais itens comprar e em que quantidade. A forma mais simples, de organizar a variedade de coisas que você precisa, é através de uma lista de compras, não é mesmo?

    As listas são fundamentais quando se fala em estruturas de dados. Elas são estruturas formadas por um conjunto de elementos encadeados, sequencialmente em nós, podendo cada um deles conter dados como variáveis e muitos outros. A vantagem desse tipo de estrutura é que numa lista os elementos podem ser acessados diretamente, sem precisar obedecer uma estrutura de filas e pilhas, sendo muito útil quando é necessário um acesso direto às informações contidas na estrutura. Como exemplo, podemos citar o acesso às informações de leitura e escrita contidas nas memórias RAM, onde os dados podem ser acessados aleatoriamente, não sendo necessário regras de filas e pilhas.

    Recapitulando

    Na computação, há diversos tipos de estruturas de dados que podem ser utilizadas para diferentes fins. Nesta unidade, você viu como funcionam algumas das principais, que são as filas, as pilhas e, principalmente, as listas. Compreender bem esses assuntos contribuirá para solucionar problemas envolvendo armazenamento de informações, bem como um controle e organização mais eficaz dos dados ao programar um software.
