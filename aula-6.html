Decisões

Durante o nosso cotidiano, temos que tomar várias decisões. Para onde ir, o que fazer quando chegar, entre outras coisas, não é mesmo? Sendo assim, inconsciente ou até conscientemente nos estruturamos uma rotina para facilitar essa tomada decisões. Na elaboração de algoritmo não é diferente, ao desenvolver um algoritmo, muitas vezes, é necessária a utilização de rotinas para tomada de decisões, e dependendo dos dados recebidos pelo sistema, o algoritmo decidirá executar determinada função.

Mas o que seria essa “função”? De forma resumida, um conjunto fechado de comandos executados linha por linha no código, utilizando de expressões lógicas e/ou aritméticas para obter um resultado.

Uma das principais características e, também, um dos maiores motivos do uso de uma função, é a praticidade na execução de comandos. Com isso, o programador não vai precisar mais escrever todas as expressões e decisões novamente, pois uma vez definida, basta apenas executar essa função no código, assim, todo o conjunto de comandos será executado novamente.

Essas decisões podem ser simples ou compostas e são tomadas utilizando os conceitos dos operadores lógicos em conjunto com os operadores relacionais que estudamos anteriormente, para decidir se uma determinada etapa do algoritmo deve ser executada ou não.

Nas decisões simples, caso a condição seja atendida, significando que obteve um resultado verdadeiro, uma função será executada; mas caso não seja atendida e obtenha um resultado falso, o algoritmo não vai executar nada, e assim, continuará o andamento do programa, acompanhe no exemplo a seguir.

Com as decisões compostas, mais de uma ação pode ser executada caso a verificação seja verdadeira ou falsa, por exemplo, podemos criar um algoritmo onde o professor insira as notas dos alunos e, então, poderá ser implementada uma rotina que decida executar uma entre duas funções: Aprovado ou Reprovado.

A primeira informa ao professor que o aluno foi aprovado caso sua média seja maior ou igual a sete (digamos que esse seja o caso verdadeiro); caso contrário (ou seja, caso seja falso), será executado a segunda função e o sistema informa ao professor que o aluno foi reprovado, assim como demonstrado no exemplo a seguir.

Independentemente de serem algoritmos para tomada de decisões simples ou compostas, todos eles possuem comandos condicionais, que permitem determinar qual função será executada de acordo com as condições preestabelecidas na programação do algoritmo.

Em programação, esses comandos servem para controlar o que deve ser executado ou não, dentro da lógica de programação. A seguir, conheça cada um deles utilizando a linguagem C# como exemplo.

IF

O comando if (se) é uma estrutura de decisão simples que usamos no nosso dia a dia, é algo tão natural que não damos conta que estamos fazendo uso de algo tão útil na área de programação, por exemplo: “se chover, não saio de casa” ou, “quando a água ferver, então desligo o fogo”, esse “se” ou “quando” que sempre usamos é exatamente o que o if representa na programação.

Ele é um comando de decisão que examina se uma determinada condição é verdadeira ou falsa, para que uma ou um conjunto de ações sejam tomadas, utilizando parênteses “( )” para delimitar qual expressão lógica será examinada e chaves “{ }” para delimitar quais ações serão executadas (que é o “escopo” do comando).

Else

O else (senão) é um comando que auxilia outra estrutura de decisão, permitindo uma segunda opção. Imagine que o if que vimos anteriormente, determinasse algo que você planejou fazer, mas alguma coisa sai errada, e agora? Nesse caso, o else seria o seu “plano B”, tendo as ações dentro do seu escopo sendo executadas assim que a condição anterior não é atendida. É importante salientar que o else só pode ser usado se já houver um comando de decisão anterior, ou seja, deve sempre estar depois de uma checagem de condição, pois, caso a condição não seja atendida, o else será executado automaticamente em seguida.

Else If

O else if (senão se) é uma estrutura de decisão formada pela junção do else e do if, assim como o else, é necessário que haja outra decisão antes, pois só é executado caso a primeira condição seja falsa. Porém, nesse momento, é possível examinar expressões lógicas entre os parênteses, como acontece no if, contanto que sejam diferentes das examinadas no comando anterior. Logo, o else if testa se uma segunda condição é verdadeira ou falsa.

Switch case

As funções if e else funcionam muito bem para casos onde se deseja testar uma ou mais condições. Mas, geralmente, nos programas são necessárias várias rotinas sequenciais, utilizando vários if e else. Além de deixar o código bastante poluído, a probabilidade do programador se atrapalhar, em algumas dessas condições, é bem considerável. Então, existe uma forma mais simples de representar um conjunto de if e else, esse comando é o switch case (também entendido como comutação).

Essencialmente, o conteúdo de uma variável é comparado com um índice de valores. Caso a comparação seja verdadeira, a rotina correspondente é acionada. Para facilitar a compreensão, imagine que criamos uma variável do tipo int, o switch case analisamos o valor dela e, para cada valor, podemos determinar várias ações.
