Programa de Computador

Um programa de computador nada mais é do que um conjunto de instruções que são utilizadas para descrever uma determinada tarefa a ser realizada pelo computador. Essas instruções são feitas de uma forma padronizada através de comandos escritos em uma linguagem de programação.

Quando você quiser executar um algoritmo em um computador, você deve escrevê-lo de uma forma entendível por ele, dando origem a um programa. Portanto, um programa de computador pode ser entendido como um algoritmo escrito em uma determinada linguagem de programação.

O programa de computador surgiu antes mesmo da invenção do computador eletrônico. Foi em 1843, quando Ada Lovelace, propôs um programa para resolver um problema matemático utilizando uma máquina inventada por Charles Babbage.

O processo de construção de um programa de computador envolve algumas etapas até que ele possa ser compreendido pelo computador. Esse é o assunto que você verá na próxima seção.

Conversão

Para que um programa de computador possa ser executado, ele deve passar por algumas etapas de conversão. Isso é necessário, pois na realidade o computador só processa as informações em bits, ou seja, em 0 e 1. O problema é que, para nós humanos, programar utilizando apenas a notação binária é muito trabalhoso e demandaria muito tempo, embora não fosse impossível. Assim, foram criadas as linguagens de programação, que possuem instruções muito próximas da linguagem natural. Por isso, é necessário possuir um mecanismo que converta essas instruções em alto nível até a codificação binária. Esse processo de conversão recebe o nome de tradução e pode ser feito de duas formas: interpretação e compilação.

No processo de compilação, o código-fonte é escrito em uma linguagem de alto nível e enviado a um programa chamado compilador, que será o responsável por efetuar a tradução.

Na compilação existe um processo intermediário entre a fase de front-end e back-end, denominada montagem. Nessa etapa, o código-fonte é traduzido para uma linguagem intermediária, chamada de linguagem de montagem. Isso ocorre, porque um único comando em linguagem, de alto nível, pode gerar diversas instruções em linguagem de máquina. Além disso, o montador também escreve as instruções necessárias para que a memória possa ser alocada para aquele programa quando for executado. Uma outra vantagem de ter a fase de montagem é que você pode utilizar uma mesma linguagem de alto nível para diferentes sistemas operacionais. Por exemplo, você pode programar ou utilizar um mesmo código-fonte no Windows e no Linux, embora as instruções de máquina para esses dois sistemas sejam diferentes. Como isso é possível? De uma forma geral, o montador vai gerar um código intermediário, denominado Assembly, com as instruções em formato de mnemônicos (pequenos textos que são “apelidos” de instruções em código binário). A partir do código em linguagem de montagem, os mnemônicos são substituídos por seus códigos binários, dependendo do sistema operacional utilizado.

Por fim, o último processo na etapa de compilação é a ligação. Nessa etapa, o ligador vai unir um ou mais código-objeto, atribuindo endereços de memória para todas as referências e gerando o arquivo executável.

O segundo processo de tradução é chamado de interpretação. Nesse caso, não é necessário gerar um código-objeto intermediário ou um executável. Assim como na compilação, a interpretação também traduz o código-fonte, mas com algumas diferenças. Na compilação, o compilador lê todas as informações, faz todas as análises e os demais processos para, no final, gerar um código-objeto ou executável. Já na interpretação, esses intermediários não são necessários e a tradução é feita de forma instantânea. Por exemplo, se você estiver programando na linguagem Python, quando você chamar o interpretador via linha de comando, ele irá ler seu código linha a linha em tempo de execução. Isso significa que, no momento em que o interpretador lê uma linha, ele traduz para a linguagem de máquina e já executa aquela linha.

O código-fonte é enviado ao interpretador, que possui um módulo de análise (léxica, sintática e semântica) e em seguida é armazenado em uma estrutura interna. O módulo de execução percorre essa estrutura interna executando as operações e consumindo os dados. Por fim, os resultados são enviados para quem chamou o interpretador.

Linguagens de Programação

Uma linguagem de programação é um conjunto de regras sintáticas e semânticas que tem por finalidade a implementação de um algoritmo que possa ser executado por um computador. Com esse conjunto de regras, o programador é capaz de especificar, de uma forma precisa, quais os dados que o computador irá atuar, como esses dados são armazenados ou transmitidos e, também, as ações que o computador deve tomar, dependendo dos dados informados.

Semântica

As linguagens de programação trazem duas abordagens para a definição se um determinado comando ou função está correto ou não: a sintaxe e a semântica. A sintaxe define a estrutura gramatical da linguagem. Veja o seguinte exemplo:

) + 10 - 5 (

Nesse caso, há um claro erro de sintaxe, pois os delimitadores (parênteses) estão posicionados de forma incorreta. Agora observe o próximo exemplo:

3 + x = false

Aqui, temos um problema semântico, pois do ponto de vista sintático todos os operadores estão sendo utilizados de forma correta, porém espera-se que o operando após o operador de atribuição (=) seja um número e não um valor-verdade.

Indentação

Nas linguagens de programação, a indentação (recuo) é um termo aplicado a um código-fonte para definir ou ressaltar a estrutura de um algoritmo. Na grande maioria das linguagens, a indentação é empregada para estruturar o código, facilitando a legibilidade. Em outras linguagens, como o Python, a indentação é utilizada para marcar a hierarquia nos blocos de comando. Embora na maioria das linguagens seja opcional, é altamente recomendável que você utilize indentação em seu código. Isso ocorrerá quando você utilizar comandos aninhados, ou seja, um comando dentro de um bloco de um outro comando. Assim, para os comandos posicionados em um nível hierárquico menor, você deve adicionar recuos (geralmente utilizando a tecla tab).

A forma indentada melhora muito a legibilidade do código, pois é possível, somente de olhar, perceber que os comandos das linhas 2, 3 e 4 estão hierarquicamente subordinados ao comando da linha 1 (recebendo um recuo) e o comando da linha 5 está subordinado ao comando da linha 4 (dois recuos).

Modularização

O processo de construção de um software pode ser feito de forma modularizada, ou seja, dividindo várias partes do sistema em partes menores que executam tarefas mais específicas. Com isso, há um ganho na manutenção do código, além de facilitar a legibilidade. Portanto, a modularização é o conceito de dividir o seu código em blocos que se comunicam entre si, com responsabilidades e tarefas específicas. Assim, você deve pensar o seu código de forma que as partes que serão reutilizadas em outros momentos possam ser escritas em blocos separados.

Documentação

A documentação de uma linguagem de programação é a maneira como se apresenta e, especifica o funcionamento, bem como os recursos de uma linguagem. Na documentação estão presentes explicações e às vezes exemplos de todos os recursos e comandos da linguagem. Assim, quando você estiver aprendendo uma nova linguagem, você deve consultar sua documentação, pois ela é a base de conhecimento e também a sua fonte de pesquisa, sendo mais rápido você encontrar a informação que você procura.

No geral, a documentação é escrita de forma mais pontual e específica, sendo dividida por tópicos, facilitando assim o processo de consulta.

Níveis de Linguagens de programação

As linguagens de programação podem ser divididas em dois níveis: alto e baixo. A grosso modo, podemos entender as linguagens de alto nível como sendo aquelas que estão mais próximas da linguagem humana. Seus comandos são escritos através de palavras em língua natural (tradicionalmente o inglês), como por exemplo o if, for, while, etc.

As linguagens de baixo nível são aquelas que estão próximas da linguagem de máquina (binário). Podem ser as linguagens de máquina em si ou as linguagens de montagem (também chamadas de linguagens de nível intermediário). Essas linguagens oferecem uma sintaxe distante de uma linguagem natural, mas muito próxima da linguagem de máquina.

Algumas Linguagens de programação

Para você conhecer melhor como é uma linguagem na prática, vamos dar uma introdução mais completa a linguagem C#, que é uma das mais utilizadas. Ela é relativamente fácil de programar e de fazer a manutenção nos códigos, uma linguagem poderosa e flexível. Com ela, é possível fazer aplicações comerciais, científicas e até jogos. Conheça os tipos de variáveis mais usados em C#, eles são:

Tipo Descrição Exemplo
int Representa os número inteiros 0 -8 254 -65
double Representa os número reais 0.4 5.98 -7.1
String Representa um conjunto de caracteres literais “Casa”
”#SENAIPLAY”
“4687”
bool Representa valores booleanos ou lógico true
false

O tipo String é imutável, ou seja, qualquer mudança em uma string causa a criação de um novo String, isso ocorre para otimizações em tempo de compilação e mais segurança, entre outros fatores

Outro tipo bem usado é o float, que é bastante semelhante ao double, mas é capaz de guardar números menos extensos, pois possui uma menor cadeia de bytes, porém um benefício de seu uso é que requer menos memória. O tipo float possui uma característica própria, em que o valor atribuído precisa ter um f no final, pois senão ocorrerá um erro, pois esse sufixo o diferencia do tipo double, por exemplo 5.45f ou 0.25f.

A nomenclatura das variáveis em C# pode ser qualquer combinação entre os caracteres válidos, que são:

  • Todas as letras (com exceção do Ç) sem acentos, e ainda independente de ser caixa alta ou baixa;
  • Todos os números, contanto que não seja o primeiro caractere do nome;
  • E o caráter especial “ _ “ conhecido como “underline” ou “underscore”.

    • A ordem dos caracteres no nome de uma variável também importa (pois dependendo da ordem dos caracteres pode formar uma palavra reservada), e no caso do uso de números, não podem ser usados para iniciar o nome.

    Mas o que seria uma palavra reservada? Em programação as palavras reservadas, são palavras utilizadas para acessar bibliotecas de dados ou comandos, que variam em cada linguagem de programação, como Java, Javascript, C, C++, C# etc, essas palavras não podem ser usadas como nome de variáveis. Utilizando o C# como exemplo, os nomes dos tipos int, float, string etc... e palavras como: class, using, public, static, entre outras… não podem ser usadas para nomear uma variável, por serem reservadas a esses comandos.

    É possível utilizar o underline para iniciar o nome de variáveis e separar palavras, exemplo: _class, _public, meu_nome, valor_menor, etc...

    Uma variável pode ser nomeada usando mais de uma palavra. Nesse caso, é recomendado que a primeira palavra seja escrita com inicial minúscula e a segunda, que precisa estar junto dela (sem o espaço entre elas), iniciada com letra maiúscula. É importante frisar que essa recomendação pode variar de acordo com a linguagem de programação utilizada.

    É fundamental usar palavras simples e descritivas que fazem referência direta para o uso daquela variável, por exemplo: se você vai usar uma variável para representar o salário de algum funcionário de uma empresa, o nome mais recomendado para essa variável seria "salario" e não “x1”, “n1”, “a”, “b”, “c”, ou qualquer outro nome não representativo.

    Após declarar uma variável, mesmo que não atribua o valor no momento, deve-se terminar a declaração com o ; (ponto e vírgula), tanto em C#, como em outras linguagens como Java, por exemplo. Veja os exemplos a seguir, para você entender melhor como deve-se nomear e não esquecer de encerrar a declaração corretamente:

    int valorMaior;
    float valorMaior;
    string valorMaior;
    boolean valorMaior;

    O código de programação precisa ser legível e de fácil manutenção, o que contribuirá para evitar erros, independente da linguagem usada. O programador deve ler o nome da variável e rapidamente compreender o que aquela variável de fato representa. Isso é importante porque o código vai crescer em número de linhas e quando for ser lido por outros programadores ou até mesmo pelo desenvolvedor, precisa estar legível.
